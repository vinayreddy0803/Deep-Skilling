Why Binary Search is usually better for sorted data:
- Linear Search checks each item one by one (O(n)).
- Binary Search halves the search range each time (O(log n)).
- For this 10-item catalog, linear search may take up to 10 comparisons.
- Binary search takes only 4 comparisons or fewer for the same catalog.
- As the catalog grows, Binary Search becomes much faster than Linear Search.

✅ Summary:
  - Linear Search: O(n)
  - Binary Search: O(log n)
  - Use Binary Search when data is sorted and random-access is available.
  - Use Linear Search when the data is small, unsorted, or when items are added frequently.
